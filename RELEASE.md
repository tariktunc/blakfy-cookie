# Release Runbook

Operator-facing guide for cutting a Blakfy Cookie release. The pipeline is
tag-driven and lives in `.github/workflows/release.yml`.

## Pre-release checklist

- [ ] All PRs for this version merged into `main`
- [ ] `CHANGELOG.md` updated with new section `[x.y.z]`
- [ ] Version bumped in:
  - [ ] `package.json`
  - [ ] `packages/cookie-next/package.json`
  - [ ] `src/api.js` `VERSION` constant
  - [ ] `dist/cookie.min.js` banner (regenerated by `npm run build`)
- [ ] All tests green on `main` (CI badge / latest `test.yml` run)
- [ ] `npm run size` passes (core ≤ 22 KB min+gzip)

## NPM_TOKEN setup (one-time)

The `publish-npm` job is fail-soft: if `NPM_TOKEN` is missing it logs a
warning and skips, so the GitHub Release still ships. To enable npm publish:

1. Login to [npmjs.com](https://www.npmjs.com/) (create the account if needed).
2. Account menu -> **Access Tokens** -> **Generate New Token**.
3. Type: **Automation** (no expiry, scoped to publish).
4. Copy the token (long string starting with `npm_`).
5. GitHub repo -> **Settings** -> **Secrets and variables** -> **Actions** ->
   **New repository secret**.
6. Name: `NPM_TOKEN`, Value: paste token.
7. Save.

Re-run the failed `publish-npm` job, or push a follow-up tag.

## Releasing v2.0.0 (or any version)

### Step 1 — Tag and push

```bash
git checkout main
git pull
git tag v2.0.0
git push origin v2.0.0
```

Pre-release tags (e.g. `v2.0.0-alpha.1`) are auto-marked as prerelease on
GitHub.

### Step 2 — GitHub Actions runs automatically

The `release` workflow runs three jobs:

1. **build** — `npm ci`, `npm test`, `npm run build`, `npm run build:next`,
   uploads `dist/` artifact.
2. **release** — creates the GitHub Release with auto-generated notes and
   `CHANGELOG.md` body, attaches `dist/cookie.min.js`, `dist/cookie.js`,
   `dist/cookie-defaults.min.js`, `dist/cookie-defaults.js`, then purges
   jsDelivr cache for each file at the new tag.
3. **publish-npm** — publishes `@blakfy/cookie-next` with `--provenance` if
   `NPM_TOKEN` is configured; logs a warning and skips otherwise.

Watch progress: `https://github.com/tariktunc/blakfy-cookie/actions`.

### Step 3 — Verify

- GitHub Release: `https://github.com/tariktunc/blakfy-cookie/releases`
- npm package: `https://www.npmjs.com/package/@blakfy/cookie-next`
- CDN (tag-pinned):
  ```bash
  curl -I https://cdn.jsdelivr.net/gh/tariktunc/blakfy-cookie@2.0.0/dist/cookie.min.js
  ```
- CDN (major-pinned, picks up latest patch):
  ```bash
  curl -I https://cdn.jsdelivr.net/gh/tariktunc/blakfy-cookie@2/dist/cookie.min.js
  ```
- Smoke test on a real site or `examples/vanilla-html.html`.

### Step 4 — Communicate

- Update `README.md` if version-specific docs changed.
- Tweet / announce if it is a major release.
- For breaking changes, link to `MIGRATION.md`.

## Hotfix / patch process

1. Branch off `main`: `git checkout -b fix/<topic>`.
2. PR -> `main`, merge after CI green.
3. Bump patch version (`v2.x.y`), update `CHANGELOG.md`, commit.
4. Tag and push: `git tag v2.x.y && git push origin v2.x.y`.
5. Pipeline runs automatically. jsDelivr `@2` will serve the new patch within
   ~12 hours, or immediately after the cache purge step succeeds.

## Rolling back

Do **not** delete tags or releases — CDN consumers may have warm caches and
deletions break reproducibility. Instead:

1. Land the fix on `main`.
2. Cut a new patch tag `v2.x.(y+1)` with the rollback / fix.
3. The `@2` major-pinned jsDelivr URL will pick up the latest patch
   automatically.

If a release was completely broken and no consumers can be on it, you may
mark the GitHub Release as **draft** (un-publish) — tags themselves should
stay intact.

## TCF v2.2 sertifikasyon

See [TCF-CERTIFICATION.md](./TCF-CERTIFICATION.md). Until certification:
`cmpId=0` (preview mode). After certification: update the assigned CMP ID
in source and bump `data-blakfy-cmp-id` in your sites' script tags.

## Troubleshooting

- **`publish-npm` skipped with warning**: `NPM_TOKEN` secret missing — see
  setup above, then re-run the job.
- **`publish-npm` 403 / E403**: token scope / 2FA — regenerate as
  **Automation** type.
- **jsDelivr still serves old file**: purge runs on release, but propagation
  can take a few minutes. Re-run the purge step or hit
  `https://purge.jsdelivr.net/gh/tariktunc/blakfy-cookie@<tag>/<file>`
  manually.
- **Release notes empty**: ensure `CHANGELOG.md` has a section for the tag
  and that `body_path` resolves; `generate_release_notes: true` provides a
  fallback from commit history.
- **Pre-release marked as latest**: include `-` in the tag
  (`v2.0.0-alpha.1`) — the workflow flips `prerelease: true` automatically.
